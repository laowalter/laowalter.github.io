Use Git

  • 删除branch
git push origin --delete gh-pages  # 删除远程库中的分支gh-pages
git branch -D gh-pages # 删除本地分支gh-pages
  • 不同的git client出现冲突，无法pull
git fetch --all  # 从远程仓库中下载最新的内容，但是不在本地合并
git reset --hard origin/master # 重置master，--hard 指全部文件
  • 删除branch
git branch -d the_local_brach  # 删除本地的
git push origin: the_remote_branch # 删除远程的
git push origin --delete the_remote_branch # 删除远程的

Deploying a subfolder to GitHub Pages Sometimes you want to have a subdirectory on the master branch be the root directory of a repository’s gh-pages branch. This is useful for things like sites developed with Yeoman, or if you have a Jekyll site contained in the master branch alongside the rest of your code.

For the sake of this example, let’s pretend the subfolder containing your site is named dist.

Step 1

Remove the dist directory from the project’s .gitignore file (it’s ignored by default by Yeoman).

Step 2

Make sure git knows about your subtree (the subfolder with your site).

git add dist && git commit -m "Initial dist subtree commit"

Step 3

Use subtree push to send it to the gh-pages branch on GitHub.

git subtree push --prefix dist origin gh-pages
