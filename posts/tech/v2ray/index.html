安装配置服务器

新服务器添加用户

  1. 添加普通用户you，并加入sudo组

     $ usermod -G sudo you

  2. Disable root 用户登录

    Filename: /etc/ssh/sshd_config
    PermitRootLogin no

  3. 启动/停止sshd服务

     $ sudo update-rc.d ssh disable
 or 
 $ sudo /etc/init.d/ssh start/stop/restart

Nginx

一切以 default 的安装为准，不做任何改变

  $ apt-get install nginx

为nginx申请let’s encrypt ssl 证书

  • 参考网址：https://certbot.eff.org/lets-encrypt/debianstretch-nginx

  • 实际申请/安装证书的过程

    1. 添加 “deb http://deb.debian.org/debian stretch-backports main” /etc/apt/sources.list 中;
    2. 执行 apt-get update;
    3. 执行 apt-get install certbot python-certbot-nginx -t stretch-backports;
    4. certbot –nginx

第三部分根据要求填写，我是直接用 yourdomain(比如 www.abcyourdomain(比如 www.abc.com)：

  • 另外根据此处的描述，也可以，但未尝试。

    Download and install Acme.sh.

    $ sudo mkdir /etc/letsencrypt
$ sudo git clone https://github.com/Neilpang/acme.sh.git
$ cd acme.sh
$ sudo ./acme.sh --install --home /etc/letsencrypt --accountemail your_email@example.com
$ cd ~
$ source ~/.bashrc

配置 nginx

服务端 Iptables rules

 1    #!/bin/sh
 2    IPT="/sbin/iptables"
 3    
 4    echo -n "Loading iptables rules..."
 5    
 6    # Flush old rules
 7    $IPT –flush
 8    $IPT –delete-chain
 9    
10    # By default, drop everything except outgoing traffic
11    $IPT -P INPUT DROP
12    $IPT -P FORWARD DROP
13    $IPT -P OUTPUT ACCEPT
14    
15    iptables -A INPUT -p icmp --icmp-type echo-request -j REJECT
16    iptables -A OUTPUT -p icmp --icmp-type echo-request -j ACCEPT
17    iptables -A INPUT -p icmp --icmp-type echo-reply -j ACCEPT
18    iptables -A INPUT -p tcp --dport 443 -j ACCEPT
19    iptables -A INPUT -p tcp --dport 80 -j ACCEPT
20    iptables -A INPUT -p tcp --dport 22 -j ACCEPT
21    
22    echo “rules loaded.”

启动客户端

  • 命令行启动

    $ v2ray -config config.json

  • Gentoo 客户端启动脚本

    Filename: /etc/init.d/v2ray
     1    #!/sbin/openrc-run
 2    
 3    depend() {
 4        need net.eth0
 5    }
 6    
 7    PIDFILE=/var/run/${SVCNAME}.pid
 8    DIR=/home/walter/.v2ray
 9    CMD=$DIR/v2ray
10    NAME=v2ray
11    OPTS="-config $DIR/config.json"
12    
13    start() {
14        ebegin "Starting v2ray ..."
15        start-stop-daemon --start --background \
16        --chdir $DIR \
17        --user $NAME \
18        --make-pidfile --pidfile $PIDFILE \
19        --exec $CMD > /var/log/v2ray.log
20        eend $?
21    }
22    
23    
24    stop() {
25        ebegin "Stopping v2ray"
26        start-stop-daemon --stop \
27        --pidfile $PIDFILE \
28        --user $NAME \
29        --exec $CMD \
30        --signal 2
31        eend $?
32    }

How to Enable TLS 1.3 in Nginx on Debian 9

  • 参考文档

  • 对于nginx编译

    1. 补充安装必要的库

       $ apt-get install pcre pcre-devel
 $ apt-get install zlib1g zlib1g-dev

    2. 采用如下配置编译

       $ ./configure --prefix=/etc/nginx
    --sbin-path=/usr/sbin/nginx
    --modules-path=/usr/lib/nginx/modules
    --conf-path=/etc/nginx/nginx.conf
    --error-log-path=/var/log/nginx/error.log
    --pid-path=/var/run/nginx.pid
    --lock-path=/var/run/nginx.lock
    --user=nginx
    --group=nginx
    --build=Debian
    --builddir=nginx-1.17.0
    --http-log-path=/var/log/nginx/access.log
    --http-client-body-temp-path=/var/cache/nginx/client_temp
    --http-proxy-temp-path=/var/cache/nginx/proxy_temp
    --http-fastcgi-temp-path=/var/cache/nginx/fastcgi_temp
    --http-uwsgi-temp-path=/var/cache/nginx/uwsgi_temp
    --http-scgi-temp-path=/var/cache/nginx/scgi_temp
    --with-compat
    --with-http_ssl_module
    --with-http_v2_module
    --with-openssl=../openssl-1.1.1c
    --with-openssl-opt=no-nextprotoneg
    --with-http_gzip_static_module
    --with-http_sub_module  //因为 site_avaliable/default中有subdir的设置，故打开此项。
    --with-stream

注意

  1. 其实通过上面的编译在 /usr/lib/nginx/modules 并没有实际生成模块文件
  2. 在nginx.conf中，是default的配置。核心内容是在site-avaliable/default中。
