步骤

建立框架

    $ hugo new site ./
    $ hugo new theme mytheme // 建立一个名为mytheme的主题

按照 https://www.zeolearn.com/magazine/develop-a-theme-for-hugo 的方法实现基本配置

处理tags/categories

tags和categories是hugo内置的两个类型，并不需要单独创建，只要文档中的front matter部分有tags和categories的内容，就可以在 http://site/tags/ 或 http://site/categories/ 列出来；

目前为止，目录结构如下：

  
  ├── archetypes
  │   └── default.md
  ├── default.md
  ├── layouts
  │   ├── 404.html
  │   ├── _default
  │   │   ├── baseof.html
  │   │   ├── list.html
  │   │   └── single.html
  │   ├── index.html
  │   └── partials
  │       ├── footer.html
  │       ├── header.html
  │       ├── head.html
  │       └── tags.html
  ├── LICENSE
  └── theme.toml

其中 tags文件用于把tag插入文章的内容中之用。

添加新的 taxomony, 例如 moods

  1. 在文档中加入此分类，比如 moods : [“happy”]

  2. 修改 config.toml，加入

    [taxonomies]
    tag = "tags"
    category = "categories"
    mood = "moods"

建立css/sass

  • mkdir asset

在theme中，

 mkdir assets

  • asset/main.scss

    $main-bg-color: #B8BDBE;
 
body {
   background-color: $main-bg-color;
}
 
h1 {
 color: green;
}

  • 修改 layouts/partials/head.html

加入 {{ style … 和 <link rel= … 内容，此时main.scss 将在引用 head.html中模板中起作用。

<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <meta http-equiv="X-UA-Compatible" content="ie-edge">
    {{ $style := resources.Get "sass/main.scss" | toCSS | minify | fingerprint }}
    <link rel="stylesheet" href="{{ $style.Permalink }}">
</head>

解析mindmap

加入如下文件

  • /themes/<theme_name>/static

    static/
├── css
│   ├── kityminder.core.css
│   └── mindmap.css
└── js
 ├── kityminder.core.min.js
 ├── kity.min.js
 ├── mindmap.js
 └── mindmap.min.js

  • /themes/<theme_name>/layout/shortcodes

    filename: themes/<theme_name>/layout/shortcodes/mind.html

    <div id="{{ .Get 0 }}" class="mindmap">
{{ .Inner | markdownify }}
</div>

    markdownify 使得 .Inner|markdown 被解析为markdown文件格式;

  • /themes/<theme_name>/layouts/partial/

    filename: /themes/<theme_name>/layouts/partial/mindmap.html

     <!-- layouts/partials -->
<head>
 <meta charset="UTF-8">
 <meta name="viewport" content="width=device-width, initial-scale=1.0">
 <meta http-equiv="X-UA-Compatible" content="ie-edge">
 
 <!-- Eable sass -->
 {{ $style := resources.Get "sass/main.scss" | toCSS | minify | fingerprint }}
 <link rel="stylesheet" href="{{ $style.Permalink }}">
 
 <!-- Insert for mindmap -->
 <script src="https://cdnjs.cloudflare.com/ajax/libs/jquery/3.3.1/jquery.slim.min.js"></script>
 <link href="{{.Site.BaseURL}}css/kityminder.core.css" rel="stylesheet" type="text/css">
 <link href="{{.Site.BaseURL}}css/mindmap.css" rel="stylesheet" type="text/css">
 <script src="{{.Site.BaseURL}}js/kity.min.js"></script>
 <script src="{{.Site.BaseURL}}js/kityminder.core.min.js"></script>

</head>

修改 Syntax Highlight

In /config.toml you need these lines:

filename: config.toml

 PygmentsCodeFences = true

For a list of styles, https://help.farbox.com/pygments.html is a good source. I think there are more, but I haven’t found a comprehensive list yet.

For the background of code blocks, actually this is set by the Hugo theme, for the Hyde theme I was using I needed to override the css like this:

filename: /themes/hyde/static/css/override.css

 pre {
   background-color: #23241f;
 }
 code {
   background-color: #EEE
 }

And add the ref link to /themes/hyde/layouts/partials/head.html

filename: /themes/hyde/layouts/partials/head.html

 ...

 <link rel="stylesheet" href="{{ .Site.BaseURL }}css/override.css">

 ...
